# Правила ведения базы знаний (Dev Notes)

Этот документ определяет стандарт оформления заметок для проектов, учебы и изучения новых технологий.

---
## 1. Главное по составлению заметок:

- **Вечное правило №1:** Заметки пишутся не бездумно и не с фулл нейрослопом - заметки нужны, чтобы мне (или заинтересовавшемуся) разобрать и понять, как работает данная тема, и насколько она завязана на других темах. Заметки могут быть не 100% точной "копиркой" с документации, но заметки написаны так, чтобы я мог понять определенные темы, описать их кратко и через время возвращаться к ним.
	
- **Нейронка будет использоваться,** но не в огромном количестве. Она нужна будет, чтобы убрать рутинную работу (структура и подобная ересь)

---
## 2. Нейминг файлов

* **Префиксы:** Каждая заметка должна начинаться с контекста.
    * *Пример:* `SQL - Оконные функции.md`
    * *Пример:* `Go - Основы горутин.md`
    * *Пример:* `PulsE - Схема авторизации.md`
    * *Пример:* `СибГУТИ - Лаба 1 (Милешко).md`

---
## 3. Структура заметки (Markdown)

### Верхний блок (Properties/Frontmatter)

- Каждая заметка начинается с метаданных (`Ctrl+;` в Obsidian):
```yaml
---
tags: [learning, sql, todo]
aliases: ["Краткое или альтернативное название"]
date: 2026-04-06
---
```

---
## 4. Оформление кода и тех. данных

- **Синтаксис:** Нужно указывать язык программирования для подсветки
```sql
SELECT * FROM users WHERE id = 1;
```
	
- **Примечания:** Callouts для акцентов:

> [!IMPORTANT] Критически важный нюанс.

> [!BUG] Описание бага, с которым столкнулся при написании кода.

---
