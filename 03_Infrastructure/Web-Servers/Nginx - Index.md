Nginx - высокопроизводительный бесплатный веб сервер, обратный прокси и балансировщик нагрузки, основанный на [[Событийно-ориентированная архитектура|событийно-ориентированной модели]]. В отличие от старых серверов по типу [[Apache - Index|Apache]], где на каждое соединение создавался отдельный процесс/поток, Nginx может обрабатывать десятки тысяч соединений одновременно, потребляя минимум оперативной памяти.

---
## 1. Основные роли

- **Веб-сервер (Static content)**: передача статичных файлов (HTML, CSS, JS) проходит намного быстрее, чем у серверов приложений
- **Обратный прокси (Reverse Proxy)**: перенаправление запросов на другие серверы
- **Балансировщик нагрузки (Load Balancer)**: распределение входящего трафика между несколькими серверами для повышения надежности и скорости работы приложения
- **Терминирование TLS**: расшифровка TLS без нагрузки на приложения

---
## 2. Структура конфигурации

В Nginx всё настраивается через текстовые файлы (обычно в `/etc/nginx/`). Основные блоки:

- **`main`**: Глобальные настройки
- **`events`**: Настройки соединений
- **`http`**: Настройки веб-трафика
    - **`server`**: Описывает конкретный домен или IP (виртуальный хост).
        - **`location`**: Определяет, что делать с конкретными путями в URL (редиректы)
