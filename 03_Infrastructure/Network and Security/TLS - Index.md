TLS (Transport Layer Security) - протокол шифрования, наследник SSL (Secure Sockets Layer). Использование протокола **https://** (HyperText Transfer Protocol Secure) гарантирует использование TLS шифрования и защиты данных посредством проверки [[Цифровой сертификат - Index|сертификатов]] сервера, с которым происходит общение клиента.

---
## 1. Основные функции:

- **Шифрование** данных и защита от прослушивания
- **Аутентификация**: гарантия того, что запрос не будет перехвачен, и клиент будет общаться именно с тем сервером, который имеет [[Цифровой сертификат - Index|сертификаты]] безопасности
- **Целостность**: гарантия того, что данные в процессе передачи никак не были изменены.

---
## 2. TLS Handshake:

Процесс установки защищенного соединения называется TLS Handshake. Он включает в себя:
1. **ClientHello**: клиент сообщает серверу о поддерживаемых версиях TLS и HTTP
2. **ServerHello**: сервер выбирает алгоритм и присылает свой [[Цифровой сертификат - Index|сертификат]]
3. **Проверка сертификата**: 
4. **Key exchange**: с помощью публичного ключа из сертификата (или с помощью [[Алгоритм Диффи-Хеллмана|Алгоритма Диффи-Хеллмана]]) стороны договариваются о сессионном ключе (симметричное шифрование)
5. **Завершение**: стороны переходят на симметричное шифрование, используя сессионный ключ для безопасной передачи данных
